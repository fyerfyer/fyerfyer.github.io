18.6.18.6.18.6.Radix Sort

1.Counting Sort

   In the previous text, we have proved that sorting using comparison has a worst case running time of Θ(nlog⁡n)\Theta(n\log n)Θ(nlogn).However, what if we don’t compare at all?

  Suppose we have a couple of characters, saying acdsjlfkmacdsjlfkmacdsjlfkm, we can sort them in alphabet order through the following steps:

  1. Creat an array with the same size of an alphabet(262626).
  2. For each character, put it in the ch−′a′ch-'a'ch−′a′ index.
  3. Scan the whole array, and the former character are always less than the latter. Insert the character into a new array in order, then we get a sorted array.

  This kind of sorting method is called counting sort. If the length of data is NNN, and the size of alphabet is RRR, then the runtime of counting sort is Θ(N+R)\Theta(N+R)Θ(N+R). It’s liner runtime!

  However, the algorithm falls to sort hard-to-count items like String, so we need improvement.
