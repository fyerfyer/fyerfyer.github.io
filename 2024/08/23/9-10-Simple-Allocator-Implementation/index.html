\(9.10.\)Simple Allocator Implementation

1.General Allocator Design

  The allocator exports three functions to application programs:

  The allocator uses the block format as below:

  The minimum block size is 16 bytes. The free list is organized as an implicit free list, with the invariant form shown as below:

  • The first word is an unused padding word aligned to a double-word boundary.

  • The padding is followed by a special prologue block, which is an 8-byte allocated block consisting of only a header and a footer. The prologue block is created during initialization and is never freed.

  • The heap always ends with a special epilogue block, which is a zero-size allocated block that consists of only a header.

  • The allocator uses a single private(static) global variable(heap_listp) that always points to the prologue block.

3.Creating the Initial Free List

  The extend_heap function is invoked in two different circumstances:

  1. When the heap is initialized and

  2. When mm_malloc is unable to find a suitable fit.

  1. Place enough space by invoking mem_sbrk.

  2. Set the new space free by setting the allocate bit 0.

  3. Set the block next to bp as the new epilogue header.

  4. Finally, in the likely case that the previous heap was terminated by a free block, we call the coalesce function to merge the two free blocks and return the block pointer of the merged blocks.

  • To maintain alignment, extend_heap rounds up the requested size to the nearest multiple of 2 words(8 bytes).

4.Freeing and Coalescing Blocks

  • The mm_free set the allocate bit of the current block to 0.
